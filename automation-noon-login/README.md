```
# Noon E-commerce Login Automation Framework

Note: Credentials are placeholder values to avoid exposing personal data. Tests demonstrate framework structure and are intended for automation design practice.

## Overview
This project is a Selenium WebDriver automation testing framework built to test the login functionality of the Noon e-commerce website using real-world scenarios.

The goal of this project is to demonstrate practical QA automation skills including test design, framework structure, and execution using industry tools.

## Tech Stack
- Java
- Selenium WebDriver
- TestNG
- Page Object Model (POM)
- DataProvider (Data-driven testing)
- Maven

## Features
- Automated login test scenarios (positive and negative cases)
- Page Object Model design for clean and maintainable structure
- Data-driven testing using TestNG DataProvider
- Explicit waits (WebDriverWait) for handling dynamic UI elements
- Assertions for validating login success and failure cases
- Real-world testing on e-commerce website (Noon)

## Test Scenarios Covered
- Valid login
- Invalid password
- Empty email field
- Empty password field
- Invalid email format
- Email case variations

## Project Structure
```

src/

```

## How to Run
1. Clone the repository
2. Open in IntelliJ IDEA or Eclipse
3. Install Maven dependencies
4. Run tests using TestNG or:
```

mvn test
